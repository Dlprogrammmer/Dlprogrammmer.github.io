网络字节序和主机字节序

网络字节序（大端字节序）：大端字节序是指一个整数的高位字节存储在内存的低地址处，低位字节存储在内存的高地址处。
主机字节序（小端字节序）：小端字节序则是指整数的高位字节存储在内存的高地址处，而低位字节则存储在内存的低地址处。
当数据在两台不同字节序的主机之间传递时，发送端总是把要发送的数据转化成大端字节序数据后再发送，而接收端知道对方传送过来的数据总是采用大端字节序。

创建socket

参数：

  1. -domain
    使用哪个底层协议
  • TCP/IP协议->(PF_INET)(PF_INET6)
  • UNIX本地域协议->PF_UNIX
  1. type
    服务类型
  • SOCK_STREAM（流服务）
  • SOCK_DGRAM（数据报服务）
  • SOCK_STREAM表示传输层使用TCP协议
  • SOCK_DGRAM表示传输层使用UDP协议
  • SOCK_NONBLOCK创建非阻塞的socket
  • SOCK_CLOEXEC用fork调用创建子进程时在子进程中关闭该socket
  1. protocol
    选择具体协议，通常设置为0.
    返回值：
    成功返回一个socket文件描述符，失败则返回-1

立即终止连接

参数：

  1. -sockfd
    待关闭的socket
  2. -howto
    shutdown的行为
  • SHUT_RD:关闭socket的读,并丢弃接收缓冲区中的数据
  • SHUT_WR:关闭socket的写,发送缓冲区数据会在关闭之前全部发送出去
  • SHUT_RDWR:同时关闭socket的读和写
    返回值：
  • 成功时返回0
  • 失败则返回-1并设置errno

TCP数据读写

参数：

  1. -sockfd
    socket文件描述符
  2. -buf
    读或写缓冲区的位置
  3. -len
    读或写缓冲区的大小
  4. flags
    额外的控制，一般为0
  • MSG_CONFIRM:数据链路层持续监听对方回应，仅用于UDP的send
  • MSG_DONTROUTE:不查看路由表，直接发送给本地局域网络内的主机，需确切知道目标主机在本地网络上
  • MSG_DONTWAIT:socket设置为非阻塞
  • MSG_MORE:告诉内核还有更多数据发送，内核可超时等待数据写入TCP缓冲区一并发送。
  • MSG_WAITALL:读操作仅在读到指定数量的字节才返回
  • MSG_PEEK:窥探读缓存中的数据
  • MSG_OOB:发送或接收紧急数据
  • MSG_NOSIGNAL:往读端关闭的socket管道或连接中写数据时不引发SIGPIPE信号

UDP数据读写

参数

  1. -sockfd

  2. -buf

  3. -len
    读写缓冲区大小
  4. -flags
    额外控制
  5. -src_addr
    发送端socket地址
  6. -dest_addr
    接收端socket地址
  7. -addrlen
    发送端或接收端socket地址长度
  • recvfrom和sendto也可用于面向连接的socket读写，只需把socket地址和长度设为null

带外标记

sockatmark判断sockfd是否处于带外标记，即下一个被读取到的数据是否是带外数据。如果是，sockatmark返回1，此时我们就可以利用带MSG_OOB标志的recv调用来接收带外数据。如果不是，则sockatmark返回0
